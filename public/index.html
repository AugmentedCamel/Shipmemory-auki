ShipMemory Bridge — Setup

First-time setup. Three short steps: create an Auki Domain, connect the bridge, pick an API key.

Step 1 — Create your Auki Domain

An Auki Domain is the storage scope where this bridge will keep your ContextCards and QR images. You need one before the bridge can do anything useful.

  1. Open console.auki.network/dashboard and sign up (or sign in).
  2. Create a new Domain. Give it any name you like — it's just for your own bookkeeping.
  3. Copy the Domain ID — it's a UUID like 2f7f0a1e-...-....
  4. Keep the email + password you used to sign up. The bridge needs them to authenticate with Auki on your behalf.

Your credentials never leave your bridge — they're stored locally in data/config.json and used only to mint Auki domain tokens.

Step 2 — Connect your Auki account

The bridge will verify these against Auki before saving. If the email/password or domain ID is wrong, you'll see an error.

Some fields are already set via environment variables. You only need to fill in the remaining ones below.
Open Auki console ↗

Paste the Domain ID from your dashboard.

Step 3 — Set a Bridge API key

✓ Set by environment variable on this deployment

Your host (Railway, Docker, etc.) already provides API_KEY. Nothing to do here — just log in with that value on the next screen.

This secret gates access to the bridge. Your client apps use it as BRIDGE_API_KEY (Mentra, Android, anything that scans your QR codes). It's also what you use to log back into this dashboard.

Pick something random — at least 10 characters. This is not your Auki password.

ShipMemory Bridge

Deploy ContextCards to your Auki Domain. Each card becomes a scannable QR code that loads a live voice agent (system prompt + tools) into a ShipMemory client — Mentra Live glasses, Android, or any app that speaks the protocol.

✓ API key is set by an environment variable on this deployment. Log in with the value you configured in your host's (e.g. Railway) environment settings.

This is the bridge's admin credential — not your Auki password.

Lost your key? Unset it on the host (edit data/config.json or the API_KEY env var) and re-run setup.
